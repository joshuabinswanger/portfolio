# UX Review ‚Äî rhizome.ch (Joshua Binswanger / Rhizome Visuals)

_Date: 2026-06-25 ¬∑ Reviewer: Claude (Opus 4.8) ¬∑ Method: live dev server (`npm run dev`) exercised with the preview tools (DOM/state/style probing + interaction), plus source tracing. Read against the per-system docs in `docs/` so by-design choices aren't flagged as bugs._

> **Note on screenshots.** I could not capture screenshots in this environment. The homepage autoplays a Mux video that **streams HLS chunks continuously and fires constant `litix.io` (Mux Data) analytics beacons**, plus loads the Google Cast SDK ‚Äî so the page never reaches network-idle and the screenshot/accessibility-snapshot tools time out waiting for a stable frame. That is itself a finding (see G-4). All findings below are instead grounded in live DOM/computed-style probing and `file:line` source traces, which is more precise for the fix list.

---

## 1. Summary

This is a **strong, characterful portfolio** with genuinely impressive engineering in places ‚Äî the archive filter/sort/zoom system and the PhotoSwipe lightbox are both well-built and hold up under testing. The visual-first intent comes through, the head/SEO layer is well above average for an artist site, and image alt text is unusually thoughtful (real scientific descriptions, not filenames).

The weak spots cluster in two areas: **(a) the two scroll-driven pages (`/fire`, `/nzz-bewerbung`) ship motion with no accessibility or lifecycle guards**, and **(b) a sitewide thinness in semantics/ARIA** ‚Äî missing landmarks, inconsistent headings, unlabeled controls, and one core interaction (project-index expand) that isn't keyboard-operable at all.

**Highest-impact issues (fix these first):**

1. **Critical ‚Äî no `prefers-reduced-motion` on `/fire` and `/nzz-bewerbung`.** Lenis smooth-scroll + GSAP pinned/scrubbed timelines run unconditionally. Only the archive zoom honors reduced-motion. (F-1)
2. **High ‚Äî scroll pages leak on navigation.** No `astro:before-swap` cleanup for Lenis/GSAP ticker, unlike the galleries; instances double-init when you revisit. (F-2)
3. **High ‚Äî project-index expand control is a non-focusable `<div tabindex="-1">`.** The primary interaction on the Index page (open a project) can't be operated by keyboard and isn't announced. (PI-1)
4. **Medium-High ‚Äî homepage never goes idle.** Autoplaying Mux video + perpetual analytics beacons + Cast SDK keep the network and compositor busy before any interaction ‚Äî a perceived-perf, bandwidth and battery cost. (G-4)
5. **Medium ‚Äî sitewide semantics gap.** No `<main>/<header>/<footer>` landmarks or skip link anywhere; heading structure is inconsistent (homepage has 10 `<h1>`, while `/info` and `/project-index` have **zero** headings ‚Äî titles are `<p>`). (G-1, G-2)

## 2. Findings by surface

Severity key: **Critical** (blocks use / accessibility violation) ¬∑ **High** ¬∑ **Medium** ¬∑ **Low** ¬∑ **Nit**.

### Global / navigation (G)

- **G-1 ¬∑ Medium ¬∑ No landmark structure.** On every page probed, `document.querySelectorAll('main|header|footer').length === 0` ‚Äî only `<nav>`. No skip-to-content link. Screen-reader/keyboard users have no landmark navigation and must tab through the whole nav on every page. _Fix: wrap page content in `<main>`, add a visually-hidden skip link, mark the nav region._
- **G-2 ¬∑ Medium ¬∑ Inconsistent heading strategy.** `/` renders 10 `<h1>` (one per project, per `GalleryElement.astro`), while `/info` and `/project-index` render **no headings at all** (titles are `<p>`). Both extremes hurt: 10 `<h1>` flattens the homepage outline; zero headings removes the document outline entirely on the other two. _Fix: one `<h1>` per page + `<h2>`/`<h3>` for sections; mark project titles on the Index as headings._
- **G-3 ¬∑ Medium ¬∑ Active nav state is JS-only and delayed.** `NavigationBar.astro:68-78` applies the active highlight inside a **`setTimeout(‚Ä¶, 500)`** on every `astro:page-load`, and there's **no `aria-current="page"`** (state is conveyed purely by color + a sliding black bar). Result: ~0.5 s with no active indicator after each navigation, and the current page is invisible to assistive tech. On a static site the active item is known at build time and should be server-rendered. (`NavigationBar.astro:139-201`, mirrored in the 3 other nav files.) _Fix: render `aria-current`/active class server-side from `Astro.url.pathname`; drop the timeout._
- **G-4 ¬∑ Medium-High ¬∑ Homepage never reaches idle (perceived perf / bandwidth / battery).** The autoplaying Mux preview opens an HLS stream and keeps pulling `.ts` chunks, fires a steady stream of `POST https://‚Ä¶litix.io/` analytics beacons, and pulls `gstatic.com` Cast SDK ‚Äî all before any interaction. This is real battery/data cost on mobile and kept the headless renderer from ever stabilizing. _Fix: gate Mux Data beaconing, lazy-init the player only when in view (an IntersectionObserver already governs play/pause ‚Äî extend it to defer load), and consider a poster-first/`preload="none"` strategy._
- **G-5 ¬∑ Medium ¬∑ Slider arrow buttons unlabeled.** Homepage gallery arrows are `<button class="ArrowLeft/Right">` with **no text and no `aria-label`** (`PortfolioGallery.astro` arrow setup ~`:303-318`) ‚Üí announced as bare "button". Project-index arrows are slightly better (literal `‚Üê`/`‚Üí` glyphs) but still read as "left/right arrow". _Fix: `aria-label="Previous image"/"Next image"`._
- **G-6 ¬∑ Low ¬∑ Generic / misspelled logo alt.** Logo `alt="Rhizom Logo"` (`NavigationBar.astro:22`) ‚Äî missing the trailing "e" (brand is "Rhizome"), and "Logo" is noise. _Fix: `alt="Rhizome Visuals ‚Äî home"`._
- **G-7 ¬∑ Nit ¬∑ Misspelled CSS custom property used sitewide.** `--font-size-dekstop-h1` (e.g. `NavigationBar.astro:288`, both `info.astro` files). It works because the typo is consistent, but it's a latent footgun. _Fix: rename to `--font-size-desktop-*` everywhere in one pass._

### Homepage gallery `/` (H)

Strong overall. Keen-slider + PhotoSwipe behave well.

- **H-1 ¬∑ Strength.** PhotoSwipe lightbox is solid: opens with `role="dialog"`, **focus moves into and is trapped in the dialog**, Close/Previous/Next carry correct `aria-label`s, and the custom caption (`photoswipeCaption.ts`) renders title + description + optional external link on image slides.
- **H-2 ¬∑ Strength.** Image `alt` text is descriptive and domain-accurate ("CT scan rendering of oak wood (Quercus robur) ~300 microns"). Keep `description_title` populated as the SEO doc warns.
- **H-3 ¬∑ Low ¬∑ `aria-modal` not set** on the PhotoSwipe dialog container (role is `dialog`). Minor; focus trap already works. _Fix: PhotoSwipe sets this in newer configs; add `aria-modal="true"` to be safe with older AT._
- **H-4 ¬∑ Nit ¬∑ Unescaped caption HTML.** `buildCaptionHTML` (`photoswipeCaption.ts:11-25`) interpolates YAML strings straight into `innerHTML`. Content is single-author/trusted so risk is negligible, but escaping (or building nodes) would be more correct.

### Project index `/project-index` (PI)

- **PI-1 ¬∑ High ¬∑ Expand interaction is not keyboard-accessible.** The row-expand affordance (`‚Üì`) is a `<div>` with `tabindex="-1"`, no `role`, no `aria-expanded`, no `aria-label`. The defining interaction of this page ‚Äî expanding a project to reveal its gallery ‚Äî is mouse-only and silent to AT. _Fix: make it a `<button aria-expanded>` (or add `role="button"`, `tabindex="0"`, key handlers, and `aria-expanded`)._
- **PI-2 ¬∑ Medium ¬∑ No headings (see G-2).** Project titles ("University Library Zurich", ‚Ä¶) are `<p>`. Add heading semantics.
- **PI-3 ¬∑ Strength.** The page is content-rich for crawlers (real titles/descriptions in static HTML) and reuses the PhotoSwipe data pattern (67 photo-links).

### Info `/info` (I)

- **I-1 ¬∑ Medium ¬∑ No headings (see G-2).** The whole About/Clients/Services page has zero headings. The "Rhizome" definition, bio, "Selected Clients", and "Services" are all non-semantic blocks.
- **I-2 ¬∑ Low ¬∑ No social / `sameAs` links.** Only `mailto:info@rhizome.ch`. For a working portfolio, links to Instagram/LinkedIn/Behance etc. are both UX and AI-SEO value (the SEO doc already lists `sameAs` as future work). Client names (ETH, Hereon@DESY‚Ä¶) are plain text, not links.
- **I-3 ¬∑ Strength.** Clear, well-written bio; clean contact path; sensible cross-link to `/personal`.

### Fire scrolly `/fire` + `/nzz-bewerbung` (F)

_(Traced in source; the scroll states themselves can't be screenshotted here.)_

- **F-1 ¬∑ Critical ¬∑ No `prefers-reduced-motion` handling.** Both pages init Lenis (`smoothWheel:true`) and GSAP `ScrollTrigger` with `pin:true` + `scrub` unconditionally (`FireScrolly.astro:35-50,90-97`; `NZZBewerbung.astro:501-509,514-519`). A repo-wide grep confirms `prefers-reduced-motion` exists **only** in `archive/filters.ts`. This is a WCAG 2.3.3 issue and a real comfort problem for motion-sensitive users. _Fix: if reduced-motion, skip Lenis and replace pin/scrub with static/stepped reveals._
- **F-2 ¬∑ High ¬∑ No lifecycle cleanup ‚Üí leak / double-init.** The galleries register `astro:before-swap` to `destroy()` slider/lightbox; the scroll pages do **not**. Lenis instances and the `gsap.ticker.add(...)` RAF handler persist across Astro view-transition navigation and re-init on return, causing scroll-interference and accumulating listeners. _Fix: destroy Lenis + kill ScrollTriggers + remove the ticker handler on `astro:before-swap`._
- **F-3 ¬∑ High ¬∑ Full scrolly runs on mobile.** The pinned/scrubbed timelines (500% on `/fire`, ~2700% on nzz) run on all viewports; only CSS/asset-resolution adapt. Janky + battery-heavy on phones. _Fix: gate the timeline behind a desktop `matchMedia` and fall back to native scroll + simple reveals on small screens._
- **F-4 ¬∑ Medium ¬∑ Keyboard scrolling hijacked.** Lenis intercepts wheel/touch/keyboard scroll; combined with `pin`, there's no skip/escape for keyboard users and a JS failure freezes the page at the pin. _Fix: tie to F-1/F-3; ensure native keyboard scroll remains usable._
- **F-5 ¬∑ Low-Medium ¬∑ `/nzz-bewerbung` is orphaned.** Not linked from any nav/footer/sitemap entry point; it's a hardcoded NZZ job-application page. Fine if intentionally unlisted, but it carries the same F-1‚Ä¶F-4 hazards and has no `noindex` consideration. _Decide: keep private (and `noindex`) or link it._
- **F-6 ¬∑ Medium ¬∑ Canvas sizing.** `FireScrolly` hardcodes a 600√ó400 canvas (`:69-70`) ‚Äî wrong scale on large/retina screens; nzz auto-sizes from the first image (better, but a layout-shift risk before load).

### Archive `/personal` (A)

The standout system. Tested live: filtering, combining, sorting, button-availability, zoom, single-column captions, mobile clamp.

- **A-1 ¬∑ Strength.** Filtering works correctly (3D ‚Üí 51/130), the active-filter **chip + `[1]` count** update, and **conflicting filters auto-disable** via the real `disabled` attribute (so they leave the tab order and empty-results states are effectively unreachable ‚Äî nice).
- **A-2 ¬∑ Strength.** Zoom is excellent: clear `aria-label`s ("Zoom in: fewer columns, larger images"), buttons disable at bounds, mobile clamps to **1‚Äì4** columns exactly as documented, and single-column toggles `is-single-column` and reveals the `<figcaption>`. The FLIP reflow honors `prefers-reduced-motion`.
- **A-3 ¬∑ Medium ¬∑ Filter buttons lack `aria-pressed`.** Active state is conveyed only by the `mixitup-control-active` class (visual). Toggle state isn't exposed to AT. _Fix: reflect active state with `aria-pressed`._
- **A-4 ¬∑ Medium ¬∑ Collapsed Filter/Sort controls stay in the tab order.** With a panel closed, its buttons are `height:0` but `visibility:visible`, `display:grid`, container `overflow:visible`, no `inert`/`hidden` ‚Äî and remain programmatically focusable. Keyboard users tab through invisible controls. _Fix: add `inert` (or `visibility:hidden`) to closed panels; remove on open._

### Personal variant `/personal` (P)

- **P-1 ¬∑ Verdict: intentional, not accidental.** Content split is deliberate ‚Äî `/personal` renders the Archive with its own nav (Gallery/Info, no Index), its own logo, and a casual bio. Good.
- **P-2 ¬∑ Medium (maintainability) ¬∑ 4√ó duplicated nav.** `NavigationBar`, `‚Ä¶Personal`, `‚Ä¶Mobile`, `‚Ä¶MobilePersonal` are ~324-348 lines each with near-identical `handleButtonClick`/`updateAnimationState`/`createEventListeners`/URL-matching logic differing only by hardcoded root path and logo. A bug fixed in one won't propagate. Also two diverged CSS-grid approaches in `Information.astro` vs `InformationPersonal.astro` (explicit `grid-row` vs `grid-auto-rows`) for the same result. _Fix: one nav component with a `variant`/`routes`/`logo` prop._

## 3. Cross-cutting themes

- **Accessibility** is the dominant theme. No landmarks/skip link (G-1), inconsistent headings (G-2), unlabeled or non-semantic controls (G-5, PI-1, A-3), focusable invisible controls (A-4), no reduced-motion on the scroll pages (F-1), and keyboard hijack (F-4). The archive shows the team _can_ do ARIA well (zoom labels, real `disabled`); the rest of the site just hasn't had the same pass. **Highest leverage: landmarks + skip link, fix PI-1, and add reduced-motion to the scroll pages.**
- **Motion & lifecycle.** The galleries clean up on `astro:before-swap`; the scroll pages don't (F-2). Standardize a teardown convention across all client scripts.
- **Perceived performance.** Eager autoplay video + constant analytics beaconing on the homepage (G-4) and full scrolly on mobile (F-3) are the two real perf costs. Both are about deferring work until it's needed/visible.
- **Consistency vs. duplication.** The `/personal` fork is intentional in content but copy-pasted in code (P-2); the misspelled CSS var (G-7) rode along through the copies. Shared components would prevent future drift.

## 5. Quick wins (low effort, high value)

1. Add `<main>` + a visually-hidden skip link, and an `<h1>` to `/info` and `/project-index` (G-1, G-2).
2. `aria-label` the gallery arrow buttons and zoom is already done ‚Äî copy that pattern (G-5).
3. Add `aria-current="page"` to the active nav link, rendered server-side, and drop the 500 ms timeout (G-3).
4. Make the project-index expander a real `<button aria-expanded>` (PI-1) ‚Äî biggest a11y win for the effort.
5. Add a `prefers-reduced-motion` short-circuit at the top of both scroll scripts (F-1).
6. Add `astro:before-swap` teardown to `/fire` and `/nzz-bewerbung` (F-2).
7. Add `inert` to closed Filter/Sort panels (A-4) and `aria-pressed` to filter buttons (A-3).
8. Bump the safe dependencies (gsap/lenis/cloudinary/sharp/marked).
9. Fix the `Rhizom`‚Üí`Rhizome` logo alt (G-6) and rename `--font-size-dekstop-*` (G-7).
